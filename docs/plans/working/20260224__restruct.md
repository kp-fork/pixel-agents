# 20260224 Restruct Plan

## Goal
- 외부 Claude 세션 추적 신뢰성을 높이면서, View 계층을 호스트 독립적으로 분리해 재사용 가능한 구조로 전환한다.

## Product Principles
- `+ Agent` 경로는 명시적이고 예측 가능한 UX를 유지한다.
- 외부 세션 자동 인식은 best-effort로 제공하되, 실패 시 사용자 개입 경로를 제공한다.
- View는 렌더링만 담당하고, 환경 의존성(VS Code/파일/터미널)은 adapter에서만 다룬다.

## Working Rules
- PR 단위 작업 규칙 문서: `docs/plans/working/20260224__restruct/20260307__pr-workflow-rules.md`

## Target Architecture
- `core`: 도메인 상태/전이 로직 (UI/호스트 의존성 없음)
- `application`: 유스케이스와 세션 매칭 정책
- `view-model`: core 상태를 UI 친화적 모델로 변환
- `view`: React/Canvas 렌더러 (입력은 view-model만)
- `adapter`: VS Code Webview/Terminal/FS/message bridge

## Epics
1. 계약 정리: 메시지/상태 스키마와 경계 타입 확정
2. View 분리: webview 내 로직을 core+view-model+view로 분해
3. Tracking 개선: 외부 Claude 세션 자동 채택 품질 개선
4. 운영 안정화: 로그/디버그 신호/회귀 시나리오 확립
5. Desktop Host 병행: VS Code extension을 유지하면서 Electrobun 기반 데스크톱 호스트를 추가

## Acceptance Criteria
- View 레이어가 VS Code API import 없이 독립 빌드 가능
- 동일 view-model 계약으로 VS Code Webview + 독립 하네스 모두 동작
- 외부 세션 인식에서 오탐은 낮추고 인식 지연은 수 초 내로 유지
- VS Code extension 기능 회귀 없이 Electrobun 호스트에서 동일 view-model 렌더링 가능
- Desktop `+ Agent` 경로에서 명령 입력 후 출력 미표시 이슈가 field repro 기준으로 해소되어야 한다.
